Ao concluir um projeto de programação de teste, direcionei minha atenção para o ambiente frontend, para explorar minhas opções para construir um aplicativo decente de forma eficiente. Inicialmente, procurei plataformas que pudessem fornecer estruturas frontend pré-construídas para acelerar parte do processo, assim como faço no desenvolvimento backend usando IA. No entanto, após experimentar ferramentas como Bildr, Softr e Versoly, percebi que o código resultante era inadequado para personalização no melhor dos casos e seria um pesadelo trabalhar depois, como tal, o que restou para mim foi buscar abordagens alternativas.

Com a ideia de engenharia reversa, comecei a olhar para a mineração de código de websites. Ao dissecar e analisar o código de um site que chamou minha atenção, eu ganharia insights sobre técnicas de desenvolvimento frontend e melhores práticas. À medida que eu vasculhava cada linha de código, a compreensão da arquitetura e design do site ficava mais clara, e também ficou claro que o frontend desse site em particular foi escrito com Vue.js, mas decidi não usar tecnologias externas como Vue, React ou Bootstrap por enquanto, e ficar com a abordagem vanilla em favor de dominar os fundamentos de HTML, CSS e JavaScript.

Enquanto minerava e polia aquele código do site, aprendi alguns truques usando CSS e Javascript para lidar com elementos HTML. Desde alternar dinamicamente a visibilidade de elementos HTML até implementar modificações de página com base no idioma e no status de login, cada passo dado dentro da mineração de código teve a oportunidade de aprender algo.

Outra funcionalidade que encontrei que vale a pena mencionar foi a minificação, o processo de remoção de todos os caracteres desnecessários do código-fonte JavaScript sem alterar sua funcionalidade. Ele cria arquivos comprimidos destinados ao consumo da máquina e expõe a importância da otimização de código no desenvolvimento web. Existem alguns sites e ferramentas que podem descomprimi-lo, como Unminify, mas você provavelmente acabará com um monte de variáveis e funções chamadas a, b, c, etc.

À medida que a necessidade de personalizar ícones e imagens se tornava aparente, comecei a fazer mais pesquisas sobre o assunto e descobri algumas dicas e ferramentas úteis como fontello, um repositório que fornece pacotes de ícones gratuitos, desenhos SVG que permitem criar designs simples com códigos, e photoshop. Essas ferramentas me permitiram personalizar ícones existentes e criar os meus próprios quando necessário para adicionar meu toque de originalidade e personalidade.

Fiz tudo o que pude para tornar meu site tão original quanto minhas habilidades permitiram na época. Como tenho planos de criar outras funcionalidades como login e gerenciamento de roteadores e outras responsabilidades a cumprir, continuarei modificando seu layout à medida que o tempo passa, aprimorando minhas habilidades de frontend para meu crescimento pessoal, e escrevendo notas do processo para projetos futuros.
